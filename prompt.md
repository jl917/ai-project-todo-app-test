## 🔍 평가 기준 (각 0~10점)

1. **정확성 (Correctness)**

   * 요구사항을 정확히 구현했는가?
   * 버그 가능성은 없는가?

2. **가독성 & 유지보수성**

   * 컴포넌트 구조가 명확한가?
   * 네이밍이 직관적인가?
   * 재사용 가능한 구조인가?

3. **성능 (Performance)**

   * 불필요한 렌더링이 발생하는가?
   * memoization(useMemo, useCallback 등) 적절한가?
   * 코드 스플리팅/지연 로딩 고려했는가?

4. **접근성 (Accessibility)**

   * semantic HTML 사용 여부
   * aria 속성 적절성
   * 키보드 접근 가능 여부

5. **UX/UI 품질**

   * 사용자 경험이 자연스러운가?
   * 로딩/에러 상태 처리 적절한가?

6. **상태 관리**

   * 상태 분리가 적절한가?
   * 불필요한 전역 상태 사용 없는가?

7. **테스트**

   * 테스트 코드 존재 여부
   * 주요 로직 테스트 커버 여부

8. **컨벤션 & 일관성**

   * 팀 스타일 가이드 준수 여부
   * CSS/스타일링 방식 일관성

* 각 항목 0~10점
* 총점 = (합계 / 80) × 100

## 🏁 승인 기준

* **90점 이상 → Approve**
* **70~89점 → Comment**
* **70점 미만 → Request Changes**

단, 아래 조건이 있으면 무조건 "Request Changes":

* 치명적인 버그
* 보안 문제
* 접근성 심각 위반
