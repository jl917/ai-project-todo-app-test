# 접근성 (WCAG 3.0 방향 · 프로젝트 가이드)

> **성격**: [WCAG 3.0](https://www.w3.org/TR/wcag-3.0/) 작업 초안 및 [한국어 개요](https://ko.htmlspecs.com/wcag-3.0/)의 구조를 **개발 습관**으로 반영합니다. 법적·조달 **공식 준수 주장**이 필요하면 [WCAG 2.2](https://www.w3.org/TR/WCAG22/) AA 등 실제 요구 규격을 별도로 충족해야 합니다.

## 우선순위 원칙

1. **WCAG 3 모델**: 핵심 요구를 기본으로 삼고, 보충 요구는 컴포넌트 라이브러리와 점진적으로 끌어올립니다. 조직적 프로세스(선언)는 문서/이슈로 추적합니다.
2. **접근성 지원**: ARIA·역할·패턴은 **프로젝트가 검증하기로 한 UA+AT 조합**에서 동작할 때만 채택합니다.
3. **범위**: 페이지/뷰 단위와 **다단계 프로세스**(결제, 온보딩 등) 전체를 함께 본다.

## 구현 시 반드시

- **의미 있는 비텍스트**: 정보 이미지에 동등 텍스트; 장식은 빈 대체·숨김 처리로 스크린 리더 노이즈 제거.
- **동기화 미디어**: 자막·대본·(필요 시) 오디오 설명; 플레이어에서 자막/설명 **켜고 끌 수 있게**.
- **단일 감각 의존 금지**: 색만, 소리만, 공간 오디오만으로 의미 전달하지 않기.
- **텍스트**: 확대·색상 대비 사용자 설정에 기능이 깨지지 않게; 보이는 문자는 DOM/대체 텍스트로 읽히게.
- **키보드**: 모든 기능 키보드로 수행; 포커스 순서가 시각적·논리 순서와 일치; 함정 없음; 모달 닫은 뒤 포커스 복귀.
- **컨트롤**: 보이는 이름과 접근 가능한 이름 정합; 상태(확장, 선택, 비활성 등)를 **시각+프로그램**으로 표시.
- **폼**: 제약·필수 항목을 **항상 보이게**; 오류는 텍스트로 설명하고 필드와 연결 (`aria-describedby` 등).
- **애니메이션**: 깜빡임·긴 모션 최소화; `prefers-reduced-motion` 존중; 트리거 콘텐츠는 사전 경고·대안 경로 고려.
- **구조**: 페이지 제목(`<title>`), 랜드마크, 제목 단계 건너뛰기 없이; 다단계는 현재 단계 표시.
- **동적 UI**: 라이브 영역 등으로 **의미 있는 변경**을 보조기술에 알림; 포커스를 억지로 옮기지 않기(예외는 명세·보안 한정).

## 하지 말 것

- `div` 클릭만으로 버튼 역할을 대체하면서 **키보드·이름·역할** 없음.
- 아이콘만 버튼에 두고 **접근 가능한 이름** 없음 (`aria-label` 또는 시각 텍스트).
- `outline: none`만 남기고 **포커스 링** 대체 스타일 없음.
- 자동 재생 오디오에 **끄기/볼륨** 없음.
- 폼 오류를 색만으로 표시.

## WCAG 2.2와 병행

- 레거시 감사·계약이 WCAG 2.x를 요구하면 **2.2 AA 체크리스트**를 별도로 통과시키고, 본 규칙은 **3.0 방향 개선**에 사용합니다.
- WCAG 2.2 AA를 충족해도 WCAG 3 Bronze와 **완전 동일하지 않을 수 있음**(명세 초안 명시).
